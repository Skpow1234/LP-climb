Raster formats (PNG, WebP, AVIF, GIF) are generated from the SVG — cost grows with width × height. Very wide presets (for example Banner) produce larger files and slower responses.

AVIF is the smallest raster option for many images, but it is slower to encode than WebP or PNG. First requests can take noticeably longer; caches make repeats fast. Smaller dimensions help a lot.

WebP defaults around quality 80–85 are usually plenty for README-style embeds — higher values mean larger files with diminishing visual return.

GIF is the slowest and heaviest option: each frame is a full SVG → raster pass. Prefer smaller width/height, fewer frames, or lower FPS when you can; use SVG or WebP for static previews when possible.
